Roadmap di Sviluppo: App Tracciamento Auto

Sviluppatore 1 = Frontend (UI, React, Mappe)
Sviluppatore 2 = Backend & Core (Capacitor, GPS, Database)

1. FASE 1: Setup dell'Architettura e Database (Fondamenta)

1.1 Sviluppatore 1: Setup Progetto Frontend

  • 1.1.1 Inizializzazione repository Git e progetto React (consigliato Vite con TypeScript).
  • 1.1.2 Installazione e configurazione di Tailwind CSS.
  • 1.1.3 Creazione della struttura base delle cartelle (es. /components, /pages, /hooks).

1.2 Sviluppatore 2: Setup Database e Motore Nativo

  • 1.2.1 Creazione progetto Supabase e design schema Database (Tabelle: users, routes).
  • 1.2.2 Installazione Capacitor nel progetto React e inizializzazione piattaforme (iOS/Android).
  • 1.2.3 Configurazione variabili d'ambiente (.env) per Supabase.

2. FASE 2: La Mappa e il Cuore del Tracking GPS

2.1 Sviluppatore 1: Interfaccia Mappa e Schermate Iniziali

  • 2.1.1 Sviluppo schermata di caricamento (Splash Screen animato).
  • 2.1.2 Integrazione libreria mappa (Mapbox GL JS). Centratura iniziale statica.
  • 2.1.3 Sviluppo UI overlay: Bottone "START", menù in alto a destra, indicatori (velocità, tempo).

2.2 Sviluppatore 2: Logica Geolocation e Calcoli

  • 2.2.1 Implementazione @capacitor/geolocation per lettura coordinate (watchPosition).
  • 2.2.2 Sviluppo algoritmi per calcolo distanza (Haversine) e velocità istantanea.
  • 2.2.3 Creazione dello State Manager per passare coordinate live al Frontend.

3. FASE 3: Logica del Percorso, Traffico e Soste

3.1 Sviluppatore 1: Rendering Dinamico della Linea

  • 3.1.1 Funzione per disegnare la Polyline sulla mappa usando l'array di coordinate in tempo reale.
  • 3.1.2 Logica cambio colore linea basata sulla velocità (Verde, Arancione, Rosso).
  • 3.1.3 Sviluppo UI per stato "In Pausa" e counter visibile.

3.2 Sviluppatore 2: Gestione Timer e Flusso Dati

  • 3.2.1 Creazione timer di partenza (Countdown 10s pre-avvio).
  • 3.2.2 Sviluppo logica Cronometri: Tempo Totale, Tempo Effettivo, Tempo di Sosta.
  • 3.2.3 Creazione struttura dati in memoria: accumulo [lat, lng, velocità, timestamp].

4. FASE 4: Chiusura Percorso, Snapshot Mappa e Salvataggio

4.1 Sviluppatore 1: Cattura Mappa e UI Resoconto

  • 4.1.1 Implementazione funzione di estrazione immagine: al click su "Fine", utilizzo di map.getCanvas().toDataURL() per generare un'immagine in Base64 del tracciato sulla mappa.
  • 4.1.2 Creazione Schermata "Resoconto Finale" per mostrare: Snapshot della mappa, Velocità Media/Max, Km, e Tempi.
  • 4.1.3 Creazione stato di "Caricamento Salvataggio" per l'attesa dell'upload sul cloud.

4.2 Sviluppatore 2: Elaborazione Dati e Storage Cloud

  • 4.2.1 Ricezione dell'immagine Base64 dal Frontend, conversione in file immagine e caricamento nel Cloud Storage di Supabase per ottenere un URL pubblico.
  • 4.2.2 Salvataggio nel Database (tabella routes) dell'URL dello snapshot assieme a tutte le statistiche finali del viaggio.
  • 4.2.3 Gestione errori di rete (es. salvataggio in local storage se offline, con riavvio automatico appena torna la connessione).

5. FASE 5: Menù, Cronologia e Utenti

5.1 Sviluppatore 1: UI Impostazioni e Storico

  • 5.1.1 Creazione UI Impostazioni.
  • 5.1.2 Sviluppo schermata "I Miei Percorsi": lista che utilizza l'URL dello snapshot generato in Fase 4 come immagine di anteprima per ogni riga.
  • 5.1.3 Sviluppo schermata "Dettaglio Storico" per rivedere le statistiche complete di un viaggio passato.

5.2 Sviluppatore 2: Autenticazione e Query

  • 5.2.1 Integrazione sistema di Login/Registrazione con Supabase Auth.
  • 5.2.2 Creazione query per il recupero dei percorsi dal DB filtrati per l'utente loggato.
  • 5.2.3 Ottimizzazione: assicurarsi che il database restituisca i dati in modo paginato per non rallentare il caricamento se l'utente ha centinaia di viaggi.

6. FASE 6: Testing, iOS Build e Rifiniture

6.1 Sviluppatore 1: Polish Interfaccia

  • 6.1.1 Controllo "Safe Area" per iOS (notch e barra inferiore).
  • 6.1.2 Aggiunta animazioni e transizioni di stato fluide.
  • 6.1.3 Supporto Dark Mode.

6.2 Sviluppatore 2: Compilazione Nativa e Background

  • 6.2.1 Configurazione del file Info.plist su Xcode per i permessi GPS (Background Location).
  • 6.2.2 Configurazione plugin per mantenere vivo il tracking GPS anche quando l'utente spegne lo schermo dell'iPhone.
  • 6.2.3 Rilascio su TestFlight per test reali in auto.
